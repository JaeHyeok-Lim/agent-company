## 1. 관찰 / 배경
개발 중 문법 오류·잘못된 CSS hex·중복 셀렉터·깨진 JSON이 여러 번 발생해 수동으로 잡았습니다.
현재 저장소엔 이를 자동으로 막는 **점검 스크립트/CI가 없습니다.** 워크플로우 JS는 일반
`node --check`로는 검증 불가(특수 하네스 형태)라 더 빠뜨리기 쉽습니다.

## 2. 문제점 (왜 안 좋은가 · 실패 모드)
변경 시 회귀를 커밋 전에 잡을 장치가 없어, 깨진 대시보드/워크플로우가 그대로 푸시될 수
있습니다. 사람이 매번 기억해서 점검해야 합니다(= 신뢰 불가).

## 3. 개선안
- **옵션 A (권고)** — `npm run check` 추가: ① 모든 JSON `JSON.parse` 검증, ② `dashboard/*.js`·
  `scripts/*.mjs` `node --check`, ③ 워크플로우는 `export const meta` 제거 후 AsyncFunction로
  파싱(하네스 형태 검증). 한 스크립트로 묶음.
- **옵션 B** — 위를 pre-commit hook 또는 GitHub Actions로 자동화.

## 4. 개선 시 비용·리스크
작음(스크립트 1개 ~40줄). 리스크 없음(읽기/검증만). B는 추가 설정 필요.
