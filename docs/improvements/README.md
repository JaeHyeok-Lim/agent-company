# 개선 결재함 (Improvement proposals)

auditor(감사팀)가 시스템 자체를 감시하며 올린 **개선 결재 서류** 목록입니다.
상태 흐름: **제안** → 승인 / 보류 / 반려. (서류 양식: [`_template.md`](_template.md))

> auditor는 코드를 직접 고치지 않습니다 — 여기에 제안만 올리고, CEO(당신)가 결재합니다.
> 승인된 건은 별도 작업으로 반영하고 상태를 **승인(반영)** 으로 갱신하세요.
>
> 📌 **규칙**: 모든 결재 서류는 맨 위에 **자연어 한국어 한줄 요약**(무엇을·왜 승인받는지)으로
> 시작합니다 — 자세히 안 읽어도 한눈에 판단할 수 있도록.
